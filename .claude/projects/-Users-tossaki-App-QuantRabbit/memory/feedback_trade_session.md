## 一言キーワード: 「トレード開始」

ユーザーが「トレード開始」「スキャルプ開始」「トレードして」等と言ったら、以下を**即座に実行**:

### Step 1: 準備（メインエージェント自身が実行）
1. `memory/project_trading_strategy.md` を読んで手法ルール確認
2. `docs/CLAUDE_TRADE_LOG.md` の直近セッションの学びを確認
3. 口座状態確認（NAV, margin, open trades）

### Step 2: マルチエージェント並列起動（3つ同時に起動）

**Agent 1: 市場分析エージェント（バックグラウンド）**
- H1/M5/M1/H4のローソク足取得
- テクニカル計算: EMA8/21, RSI, BB(20,2σ), ATR, フィボナッチ
- レジーム判定: TREND(up/down) / RANGE / TRANSITION
- エントリーゾーン特定（BB端、フィボレベル、S/Rレベル）
- 結果をメインに報告

**Agent 2: 記録エージェント（バックグラウンド）**
- `docs/CLAUDE_TRADE_LOG.md` にセッション開始情報を追記
- トレード完了のたびにSendMessageで結果を受け取り記録
- 思考ログも記録

**Agent 3: 相場別トレードエージェント（フォアグラウンド）**
- 分析エージェントの結果を元に、レジームに合ったトレードを実行
- レジーム変化時はエージェントを切り替え

### Step 3: レジーム別エージェント切替

| レジーム | 判定条件 | エージェント | 手法 |
|----------|----------|-------------|------|
| **TREND(down)** | EMA gap<-2pip or M5ベア3本+ | トレンドフォロー | 戻り売り（M1反発→再下落で入る） |
| **TREND(up)** | EMA gap>+2pip or M5ブル3本+ | トレンドフォロー | 押し目買い（M1下落→再上昇で入る） |
| **RANGE** | EMA gap<1pip, ADX<25 | レンジスキャルプ | BB端逆張り + ATR×50%即利確 |
| **TRANSITION** | 上記どれにも該当しない | 待機 | エントリーしない。監視のみ |
| **低ボラ** | M1 ATR<0.8pip | 超短期 or 休止 | スプレッドコスト>利益になるため基本休止 |

レジームは**M5キャンドル完了ごとに再判定**。変化したら手法を即切替。

## 必須ファイル
- `scripts/claude_scalp_loop.py` — OANDA API ヘルパー関数群
- `docs/CLAUDE_TRADE_LOG.md` — トレード記録・思考ログ・手法ルール
- `config/env.toml` — OANDA APIキー
- `indicators/calc_core.py` — 54種のテクニカル指標エンジン
- `indicators/factor_cache.py` — M1/M5/H1/H4/D1キャッシュ

## 利用可能テクニカル（indicators/calc_core.py）
トレンド系: MA, EMA, ADX, DI, EMA_Slope / モメンタム系: RSI, MACD, Stoch_RSI, CCI, ROC
ボラ系: ATR, BB, Keltner, Donchian / S/R系: Swing距離, Cluster / 一目均衡表 / ダイバージェンス / VWAP

## トレード管理ルール（全レジーム共通）
- SLなし、手動管理（SL地獄回避）
- 打診(1500-2000u) → 確認 → 本玉追加(1000u) ※合計4000u以下
- ナンピンは1回まで、-40 JPYで損切り
- 利確: ATR×50%以上 or +15 JPY以上で即利確
- 5/5連続同方向M1 = トレンド末期、エントリーしない
- 待機時間: ボラ高→5秒、通常→M5完了ごと
