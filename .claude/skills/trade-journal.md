# トレード日記スキル

トレードの振り返りを構造化して記録する。学習ループの核。

## 使い方

- 「日記」→ 直近クローズしたトレードの振り返り
- 「今日のレビュー」→ 今日の全トレードを統計+振り返り
- 「週次レビュー」→ 今週の全トレードを統計+パターン分析

## 実行手順

### Step 1: トレードデータ取得

1. OANDA API: `/v3/accounts/{acct}/trades?state=CLOSED&count=20` で直近クローズ済みトレード
2. `logs/live_trade_log.txt` からエントリー時の判断根拠を検索
3. `logs/shared_state.json` からクローズ時のレジーム（参考）

### Step 2: 個別トレード振り返り（対話形式）

各トレードについて以下を分析・記録:

```
## 📝 トレード日記: [ペア] [方向] [日時]

### トレードサマリ
| 項目 | 値 |
|------|-----|
| ペア | EUR/USD |
| 方向 | SHORT |
| Entry | 1.14755 |
| Exit | 1.14350 |
| P&L | +xxx JPY |
| 保有時間 | 2h 15m |
| R:R実績 | 1.8:1 |

### エントリー判断
- **根拠**: (live_trade_logから)
- **Play**: H1 Bear + M5 Bounce Peak
- **コンフルエンス**: H4 bear + H1 MACD hist neg + M5 StochRSI peak
- **タイミング**: ⭐良い / 🔶やや早い・遅い / ❌悪い

### イグジット判断
- **クローズ理由**: TP到達 / SLヒット / 裁量クローズ / BE停止
- **タイミング**: ⭐良い / 🔶早すぎ・遅すぎ / ❌悪い
- **利確の妥当性**: (TPまで到達した？ / 途中で反転した？)

### 改善点
- **良かった点**: xxx
- **悪かった点**: xxx
- **次回への教訓**: xxx

### タグ
#勝ち or #負け, #トレンド継続 or #逆張り, #スキャルプ or #スイング, #感情的 or #計画的
```

### Step 3: 統計（日次/週次の場合）

```
## 📊 統計サマリ

| 指標 | 値 |
|------|-----|
| トレード数 | N |
| 勝率 | xx% |
| 平均勝ち | +xxx JPY |
| 平均負け | -xxx JPY |
| P&L合計 | +/-xxx JPY |
| 平均R:R | x.x:1 |
| 最大勝ち | +xxx JPY |
| 最大負け | -xxx JPY |
| 平均保有時間 | xh xm |
| Profit Factor | x.xx |

### パターン分析
- **最も勝てたPlay**: xxx
- **最も負けたPlay**: xxx
- **時間帯別勝率**: 東京=xx%, ロンドン=xx%, NY=xx%
- **ペア別P&L**: UJ=+xx, EU=-xx, ...

### 失敗パターン検知
⚠️ 「ロンドン開場後のSHORTで3連敗」
⚠️ 「BE停止後の再エントリーで2回負け」
⚠️ 「RSI<30での逆張りLONGが0勝3敗」
```

### Step 4: 保存

振り返り結果を `docs/trade_journal/YYYY-MM-DD.md` に保存。
失敗パターンは memory の `project_trading_strategy.md` に追記。

## ルール

- **正直に書く**。勝ちトレードでも「たまたま」なら書く。
- **感情も記録**。焦り・恐怖・欲張りがあったら隠さない。
- **必ず「次回への教訓」を1つ以上**。振り返りなしの記録は無意味。
