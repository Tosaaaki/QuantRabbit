# 逆説チェック（Devil's Advocate）スキル

トレードアイデアの反対側の根拠を強制的に列挙する。確証バイアスを破壊するツール。

## 使い方

ユーザーが「EU SHORT 逆説」「UJ LONGチェック」等と言ったら実行。

## 実行手順

### Step 1: 現在のデータ取得

OANDA APIから対象ペアの価格を取得し、factor_cacheまたはshared_state.jsonからテクニカルを取得。

### Step 2: 反対根拠の列挙

ユーザーが考えている方向の**反対方向**を支持する根拠を、以下の全カテゴリから洗い出す:

#### A. テクニカル反対根拠
- **MTF矛盾**: H4/H1/M15/M5で反対方向を示す時間軸はあるか
- **RSI/StochRSI**: 売られすぎ/買われすぎでの逆張りシグナル
- **ADX弱体化**: トレンドが弱まっている兆候
- **ダイバージェンス**: 価格とオシレーターの乖離
- **キーレベル**: フィボ/S&R/VWAPでの反転リスク
- **Ichimoku**: 雲のサポレジ、転換線/基準線のクロス
- **ボリンジャー**: バンドウォーク終了兆候、スクイーズ

#### B. マクロ/ファンダ反対根拠
- **金利差**: 中銀スタンスの方向
- **経済指標**: 直近発表の影響
- **地政学**: リスクオン/オフの文脈
- **フロー**: 月末リバランス、オプション満期

#### C. 構造的反対根拠
- **セッション**: 東京/ロンドン/NY切替での方向転換リスク
- **流動性**: 薄い時間帯での逆行リスク
- **ポジショニング**: 片側に偏りすぎていないか
- **相関**: 他ペアとの矛盾

#### D. 過去の失敗パターン
- `docs/TRADE_FINDINGS.md` や memory の lessons から類似失敗を検索

### Step 3: 出力フォーマット

```
## 🔴 逆説チェック: [ペア] [方向]

### あなたの根拠（推定）
- xxx
- xxx

### 反対根拠
| カテゴリ | 根拠 | 重要度 |
|----------|------|--------|
| テクニカル | H1 RSI=28 売られすぎ反発リスク | ⚠️高 |
| マクロ | BOE利上げ期待でGBP買い圧力 | 🔶中 |
| 構造 | ロンドン開場で方向転換しやすい時間帯 | 🔶中 |
| 過去失敗 | 3/18 同セットアップで-15pip | ⚠️高 |

### 総合判定
❌ 反対根拠が強い → エントリー見送り推奨
⚠️ 反対根拠あるが限定的 → SL広め/ロット減で対応
✅ 反対根拠が弱い → エントリーOK

### 改善提案
（SL調整、タイミング変更、別ペア提案等）
```

## ルール

- **必ず3つ以上の反対根拠を出す**。「問題なし」は禁止。何か見つけろ。
- 根拠はデータベースで。「なんとなく」禁止。
- ユーザーの方向を否定するのが目的ではない。盲点を見つけるのが目的。
