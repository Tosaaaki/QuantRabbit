# 変更時の必須プロトコル

コード・設定・アーキテクチャを変更したら、以下を全て実行:

1. **CLAUDE.md更新**: アーキテクチャ変更時はルートのCLAUDE.mdを更新
2. **メモリ更新**: 該当するメモリファイルを更新
3. **変更ログ追記**: `docs/CHANGELOG.md` に追記
4. **mainにマージ**: ワークツリー編集時は必ずmainにマージ
5. **即デプロイ**: 変更したら即反映。聞くな
6. **日英同時編集**: プロンプトファイル（.claude/rules/、CLAUDE.md、scheduled-tasks/*/SKILL.md）を編集する際は、英語版（運用）と日本語版（参照用）を必ず同時に更新。内容は常に同期させる

## 言語ルール — トークンコスト最適化

**コンテキストウィンドウに入るものは全て英語。** 日本語は同じ内容で英語の2-3倍のトークンを消費する。

| 対象 | 言語 | 理由 |
|------|------|------|
| プロンプトファイル (.claude/rules/, CLAUDE.md, SKILL.md) | **英語** | 毎セッション自動ロード = 高トークンコスト |
| ツールスクリプト出力 (.pyのprint/f-string) | **英語** | Claudeが読む = トークンコスト |
| state.md、trades.md、内部メモ | **英語** | 毎セッション読み書き = トークンコスト |
| Slackメッセージ（ユーザー向け） | **日本語** | ユーザーがSlackを読む。コンテキストに入らない |
| 日本語参照コピー (rules-ja/, CLAUDE_ja.md, SKILL_ja.md) | **日本語** | ロードされない。ユーザー確認用 |

**新規作成・修正時:**
- 新規ツールスクリプト → 出力文字列は全て英語
- 新規プロンプトファイル → 英語で書き、日本語コピーを作成
- 新規定期タスク → SKILL.mdは英語、SKILL_ja.mdは日本語
- Slack通知 → 日本語（唯一の例外）
