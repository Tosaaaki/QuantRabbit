## スキャルプ実行の3大失敗 (2026-03-19 -795JPY損失から学習)

**Rule:** スキャルプは2-4pip即利確。+2pip乗ったら即BE/trail。サイズ1500u上限。

**Why:** 2026-03-19に30トレードで-795JPY。原因:
1. **利確が遅い**: +594JPYのUPLがあったのにTP 6-8pipを狙って全額吐き出し
2. **サイズ過大**: 2300u, 3500uで入って反転時のダメージ甚大
3. **リベンジトレード**: 同じペアにSL後即再エントリーして累積損拡大（EUR_USD -300以上）

**How to apply:**
- TP: UJ/EU/AU=3pip, GU/EJ/AJ=4pip, GJ=5pip。欲張るな
- +2pipでBE、+3pipでtrail 1.5pip
- 1500u上限。Opusの裁量判断で「伸ばすべき」と思っても1500u
- SL後の同一ペア10分クールダウン、3連敗15分停止
- **ルールで縛るのはボット化。しかしサイズと利確だけはルール**。方向判断は裁量

### 追加教訓: Sonnet vs Opus
- Sonnetでは市況の深い読みができず、パターンマッチングに堕落した
- scalp-fastをOpusに変更（コスト増だが裁量判断の質が本質）
- ルール追加より思考の深さを上げるのが正しいアプローチ
