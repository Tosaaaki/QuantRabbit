**道具は腕の延長。選び・研ぎ・育てろ。頭がボットになるな。**

**Why:** Claudeは道具を「使うか使わないか」の二択で考えがち。また提案が機械的（条件→自動実行）になりがち。ユーザーが求めているのは、道具を自分の手足として育てながら、市況を読んで判断するプロトレーダー。(2026-03-20)

**How to apply:**
1. **道具との関係**: OANDAもmonitorもスクリプトも全てClaudeの道具。使う/使わないの二択ではなく:
   - 選ぶ: 市況を見て今日はどの道具をどう使うか
   - 調整する: パラメータ、registry設定をポジションごと・局面ごとに変える
   - 改良する: コード自体を書き換える。計算方法を変える
   - 作る: 必要な道具がなければ作る
   - 捨てる: 使えない道具は捨てる
2. **頭のボット化チェック**: 「なぜこの判断？」に市況で答えられるか。「スコアが高い」「条件が揃った」は不合格
3. **ルール・スコア・チェックリストは参考**: 今の市場に合っていなければ無視してよい
4. workers/常駐ボットは禁止。注文はOANDA REST API直接（urllib経由）
