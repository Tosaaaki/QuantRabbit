## 最重要: 俺自身がボット脳ループに陥る

**パターン:** 問題を見つける → 具体的な数字/閾値で修正 → traderがそれを機械的に従う → また同じ問題 → …

**実例（2026-03-20）:**
- 「TP=75pipは東京で遠すぎる」→ 反射的に「東京=TP 5-10pip」と書いた → これ自体がボット用ルール
- ユーザーに指摘されて「TPに届く根拠を1文で言えるか？」に修正

**Why:** 数字を書いた瞬間、traderは「なぜその数字か」を考えずにその数字を使う。それはルールの奴隷であり、裁量トレーダーではない。VIX=35の東京とVIX=12の東京は全然違うのに「東京=5-10pip」では対応できない。

**How to apply:**
- **数字/閾値を書きたくなったら → 問いに変換しろ。** 「TP 5-10pip」ではなく「このTPに届く根拠は何だ？」
- **セッション/条件で固定値を分けたくなったら → それも機械的ルール。** 原則を1つ書け
- **「〜するな」を書く前に → 「なぜダメか」を事例で示して判断を委ねろ**
- **ルール追加したくなったら → 手法として教える。** 判断は任せる
- テーブル/チェックリスト追加したくなったら → スケール感を伝えるだけにする
- プロンプトが500行超えたら整理タイミング
