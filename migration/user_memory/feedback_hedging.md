OANDA v20ヘッジ口座では両建てが可能。**マージン計算は大きい方の片側にフルマージン。反対側は追加マージンゼロ。**

**Why:** ユーザーからの指摘。「ショート側のマージン空いてるってことにならないの？」→ その通り。既存ポジの逆側はタダで建てられる。

**マージン計算ルール（OANDA v20）:**
- LONG 1000u + SHORT 1000u以下 → マージンはLONG 1000u分のまま（追加0）
- SHORT > LONGになると、SHORT側にフルマージン切り替え
- ファイナンスコスト（スワップ）は両側にかかる（短期回転なら無視可）

**How to apply:**
- **MTF階層分離**: H1テーゼLONG維持 + M5過熱でSHORT回転。テーゼは矛盾しない、TFが違うだけ
- **M5過熱検知 → 即ヘッジSHORT**: StochRSI≧0.95, RSI>70, CCI>200 → マージン追加0でSHORT
- **プルバック利確 → マージン解放**: SHORT利確益でAUD等の新規ペアに資金を回す
- **フリップ時**: 先にSHORT建て → 確認後にLONG決済（スリッページ回避）
- **ニュース前**: 同量ヘッジで両面カバー → 方向見えたら片方決済
- **state.mdに「M5 SHORT hedge」と明記**: テーゼの階層を記録すること
- traderのSKILL.md + strategy_memory.mdに戦術として組み込み済み（2026-03-25）
