## 構想: アラート駆動トレードシステム

Traderが事前にセットアップ条件をアラートとして登録 → live_monitor.py(30秒)が検知 → 機械的注文は即執行、裁量判断はtraderをキック。

### 設計案
1. Traderが `logs/trader_alerts.json` に条件+コンテキスト(なぜこのセットアップか、SL計画等)を書く
2. live_monitor.pyがアラート条件を30秒ごとにチェック
3. 条件ヒット時:
   - 機械的注文(指値)→ Pythonで即執行(30秒以内)
   - 裁量判断必要 → traderタスクをオンデマンドキック(1分以内)
4. 引き継ぎ: アラートJSONにコンテキスト埋め込み。次のtraderが「なぜこのアラートか」を読める

### 期待効果
- 初動遅延: 最悪3-4分 → 機械的30秒/裁量1分
- エントリー精度向上(狙ったレベルで入れる)
- 見逃し減少、リベンジトレード抑制

### 着手条件
**Why:** 速度改善は「良い判断を取りこぼさない」仕組み。判断自体が悪ければ速くても負ける。
**How to apply:** v4プロンプト改善後のトレード判断の質が安定したことを確認してから着手。まずサンプルを増やす。
